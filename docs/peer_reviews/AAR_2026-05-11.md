## 4. Root Causes (DE1: Root Cause Analysis)

**Deviation: File restored to pre-hardening state**
- Why 1: `git checkout -- docs/UNIFIED_ROADMAP.md` was executed in a prior shell operation that the user or system triggered between sessions.
- Why 2: The v2.0 hardening edits from the previous session were in the working tree but uncommitted when the session ended. [unverified: the session summary says "working tree on main had uncommitted changes" and "BUS phase and RETROGRADE phase were planned but not yet executed"]
- Why 3: No pre-session state snapshot was captured before the `checkout` command.
- Root cause: **Working tree changes were not committed before session close**, and a subsequent `checkout` discarded them. This is a process gap in session handoff.

**Deviation: Script missed 2 edge cases**
- Why 1: The script used `if "string" not in content` guards that were too conservative — they checked for the *section header* but the insertion anchor string was subtly different from what was expected.
- Why 2: The `replace_all` parameter on the `edit` tool in an earlier attempt failed because the old string was not unique, causing the script fallback.
- Root cause: **Script was written reactively after a failed `edit` tool call**, without full file content inspection. The script encoded assumptions from the failed tool output rather than the actual file state.

## 6. Improves (IN20: Antigoals & Anti-Patterns Catalog)

- **Working tree changes not committed before session handoff** — The previous session ended with uncommitted v2.0 hardening edits. When a `git checkout` ran between sessions, all edits were lost. This is a session-handoff anti-pattern. Evidence: session summary states "working tree on main had uncommitted changes"; `git diff --stat 0e77e57..HEAD` shows 131 lines changed in v2.0 commit that had to be re-applied.
- **Script was written without inspecting actual file content first** — The `_harden_roadmap.py` script encoded assumptions about string uniqueness and section ordering that did not match the restored v1.0 file. Two edge cases were missed. Evidence: manual edits at 20:41Z and 20:42Z were required after script execution.
- **No stop rule for the peer review process** — 4 reviewers were launched without a criterion for "enough review." A 5th reviewer (e.g., security/redteam) would have been valuable but was not considered. Evidence: External reviewer noted that "no reviewer was adversarial in the security sense."
- **No external auditor designated** — All 4 reviewers were agents. The Measurement reviewer explicitly flagged: "There is no measurement-independent observer anywhere in the system." Evidence: Measurement review section 4, Campbell corruption map.
- **Synthesis did not include raw review artifacts in the repo** — The raw ~14,000 words of review output were written to temp overflow files and are not preserved in the repo. Only the 2,800-word synthesis is committed. Evidence: synthesis document notes "Overflow logged to temp" for each reviewer.
