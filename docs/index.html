◉ CRAB

Coordination Receipts for Agent Behavior. A 4-step protocol that makes autonomous agents observable, accountable, and safe.

View on GitHub Get Started

The Problem

You're running multiple AI agents. They work while you sleep. But when you wake up, you have no idea what happened. Did they conflict? Did they fail silently? Did one overwrite another's work?

CRAB fixes this. Every agent turn leaves a receipt on a coordination bus. You can replay any session, audit any decision, and know exactly what happened while you were away.

The Protocol

Every autonomous turn follows four steps. Each step produces structured output. The final step posts a receipt to the bus.

C

Check

Read live state: git branch, dirty files, stashes, bus tail, blockers

R

Reason

Decide if action is needed, choose lane, pick message type, record rationale

A

Act

Perform work: cleanup stale branches, audit the bus, run a lane handler

B

Bus

Post a receipt: timestamp, identity, type, actions taken, success/failure

Install & Run

Zero dependencies. Copy one file, run one command.

$ git clone https://github.com/hummbl-dev/crab.git
$ cd crab
$ python crab_daemon.py --init
Default config written to: crab-daemon/config.json
$ python crab_daemon.py --once --verbose
=== CRAB turn: cleanup ===
CHECK: branch=main dirty=False stashes=0 blockers=0
REASON: should_act=True type=STATUS rationale=Lane cleanup ready
ACT: success=True actions=2 errors=0
BUS: posted=True

Features

Pluggable Bus

TSV, JSONL, stdout, or custom callback. Write to files, webhooks, databases, or Slack without touching the core.

Multi-Lane

Run independent work streams on different schedules. Cleanup every hour, audit every 10 minutes, custom lanes as needed.

Zero Dependencies

Python 3.8+ and the standard library. No pip install, no version conflicts, no supply chain risk.

Observable

Every turn produces a receipt. Replay any session. Know who did what, when, and why.

Safe by Default

Blockers stop lanes. Stashes pause sensitive work. Dry-run mode simulates without mutating state.

Container Ready

Single-file daemon. Drop into Docker, Kubernetes, or systemd with zero config.

Bus Backends

Choose where receipts go. Switch backends without changing your lanes.

{
  "bus": {
    "backend": "tsv",           // tsv | jsonl | stdout | callback
    "path": "bus/messages.tsv"  // file path for tsv/jsonl
  }
}

TSV (default)

Append-only tab-separated log. Human-readable, grep-friendly, works with any Unix tool.

JSONL

Structured JSON Lines output. Easy to parse programmatically, query with jq, or load into databases.

Stdout

Print receipts to stdout. Perfect for containers, systemd, or watching in real time.

Callback

Run a shell command after every turn. Send to Slack, trigger CI, update a dashboard, or call any webhook.

Use Cases

Nightly agent fleet sync. Run CRAB on a scheduler. Every morning, read the bus to see what each agent did overnight.

CI pipeline coordination. Use the callback backend to trigger downstream jobs only after upstream agents post success receipts.

Multi-machine mesh. Mount a shared TSV file via NFS or sync it with Syncthing. Every machine appends its receipts to the same log.

Audit trail for compliance. The JSONL backend produces tamper-evident logs. Every receipt includes timestamp, identity, and action taken.

API Reference

The daemon exposes a CLI and a programmatic API. Both use the same config format.

python crab_daemon.py --init              # Write default config
python crab_daemon.py --once             # Run one iteration
python crab_daemon.py --once --dry-run   # Simulate, do not post
python crab_daemon.py --lane cleanup     # Run only one lane
python crab_daemon.py --verbose         # Debug logging
python crab_daemon.py --config path    # Custom config file

Programmatically:

from crab_daemon import CrabDaemon, DaemonConfig, BusConfig, LaneConfig

config = DaemonConfig(
    identity="my-agent",
    bus=BusConfig(backend="stdout"),
    lanes=[LaneConfig(name="cleanup", enabled=True)]
)

daemon = CrabDaemon(config)
daemon.run()  # Continuous loop
daemon.run_once()  # Single iteration
